PR Creation Guidelines

How to Create Effective PR Entries

✅ Best Practices

  • Use descriptive branch names (feature/fix/hotfix prefix)
  • Include clear description of changes
  • Set appropriate priority level
  • Specify target testing environment
  • Add planned test scenarios in description

💡 Pro Tips

  • Create entries as soon as development begins
  • Update descriptions as scope changes
  • Use priority levels to help QA prioritize work
  • Include links to related tickets/issues
